Privacy & Security Help Privacy and Security Overview

This mobile inventory management application is committed to protecting user data and maintaining secure access to inventory information. Below are the key privacy and security principles integrated into the system.

1. Data Collection

No personal or sensitive user data is collected. Only inventory-related information such as item name, category, quantity, price, and supplier details is handled.

2. Data Storage and Transmission

All data communications use HTTPS to ensure encrypted and secure transmission between the app and the official REST API (https://prog2005.it.scu.edu.au/ArtGalley).

3. Access Control

While current access is unrestricted for learning purposes, the system supports secure operations with assumed backend role-based access control in production.

4. Device and App Permissions

The app does not request or access any native device features such as storage, location, or camera. It operates completely within the browser or webview.

5. Immutable Audit Fields

Specific records like "Laptop" are protected and cannot be deleted. This preserves essential data during demonstrations and supports learning integrity.

6. Data Validation

Input fields such as Quantity and Price are validated to accept numeric values only. Required fields are checked before form submission to ensure data integrity.

7. External APIs

The app communicates only with a single official API hosted by Southern Cross University. No third-party services are contacted.

8. User Consent

As the app does not handle personal user data, consent dialogs are not required. Future enhancements involving authentication will include proper consent flows.

9. App Security Practices All HTTP requests are secured with HTTPS and Angular HttpClient. UI restricts invalid operations such as deleting protected records. User-friendly error messages guide secure interaction. 10. Summary

This system is designed with student safety and responsible development in mind. It demonstrates modern app security techniques while allowing a smooth and friendly user experience.

If you have questions, contact your instructor or system administrator.
